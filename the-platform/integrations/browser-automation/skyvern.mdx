**Description:** Manage operations in Skyvern.

**Authentication:**
- See integration-specific authentication requirements

**Available Actions:**

**Examples:**

##### Action: `unknown`

**Example 1: Skyvern - List Workflows**

Retrieve a list of all workflows available in your Skyvern account.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-list-workflows-001"
}
```

**Example 2: Skyvern - Run Workflow**

Execute a Skyvern workflow asynchronously with parameters. The workflow will run in the background and return a run_id for tracking.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-run-workflow-001"
}
```

**Example 3: Skyvern - Run Workflow (No Webhook)**

Execute a Skyvern workflow without webhook notifications. Set webhook_callback_url to null to explicitly disable webhook. Omit the field to use SSM default. Use get_job_status to check progress.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-run-workflow-002"
}
```

**Example 4: Skyvern - Get Job Status by Run ID**

Check the status and progress of a specific workflow run using only the run_id.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-get-job-status-001"
}
```

**Example 5: Skyvern - Get All Runs for Workflow**

Get all runs and their statuses for a specific workflow. Returns list of all runs.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-get-job-status-002"
}
```

**Example 6: Skyvern - Poll Job Status (by run_id)**

Poll the status of a workflow run using only the run_id. Simpler than get_job_status as it does not require workflow_id.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-poll-job-status-001"
}
```

**Example 7: Skyvern - List Jobs**

List all workflow runs/jobs. Can optionally filter by workflow_id and paginate with limit/offset.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-list-jobs-001"
}
```

**Example 8: Skyvern - List All Jobs**

List all workflow runs/jobs across all workflows.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-list-jobs-all-001"
}
```

**Example 9: Skyvern - Run Workflow Synchronously**

Run a workflow and wait for completion by polling. Returns the final result when workflow completes or times out.

```json
{
  "automationType": "skyvern",
  "nodeId": "skyvern-run-workflow-sync-001"
}
```
